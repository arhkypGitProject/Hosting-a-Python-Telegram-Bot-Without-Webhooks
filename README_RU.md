* Простое развёртывание Telegram бота на Aiogram 3 на VPS с Ubuntu (AdminVPS.ru)

<b>Прежде чем мы начнём, я хочу кратко пояснить, какой хостинг-провайдер был использован и почему.</b>

### Объяснение терминов

- **Командная строка (CMD)** — интерфейс командной строки, встроенный в Windows, который позволяет пользователям выполнять команды и взаимодействовать с операционной системой с помощью текстовых команд.

- **SSH (Secure Shell)** — сетевой протокол, который позволяет безопасно подключаться к удалённому серверу и управлять им по зашифрованному соединению.

- **VPS (Virtual Private Server)** — виртуальная машина, размещённая на физическом сервере, которая предоставляет выделенные ресурсы и полный контроль над операционной системой.

Для данного руководства я использовал *[AdminVPS.ru](https://adminvps.ru)*. Я решил записать это руководство во время выполнения реального клиентского проекта. Клиент приобрёл VPS специально для размещения и защиты своего Telegram бота.

В целях конфиденциальности и безопасности вся чувствительная информация — включая IP-адрес сервера, учётные данные, доменные имена и любые данные, которые могут идентифицировать клиента или его инфраструктуру, — была скрыта на протяжении всего руководства. </b>

Сначала вам понадобится VPS-сервер. Вы можете выбрать любого хостинг-провайдера, который вам нравится, поскольку большинство VPS-услуг очень похожи, и процесс развёртывания практически идентичен у разных провайдеров.

В этом руководстве я буду использовать *[AdminVPS.ru](https://adminvps.ru)*, но вы можете выбрать любого другого провайдера VPS.

<b>После открытия страницы управления сервером вы увидите информацию, подобную этой. Она содержит IP-адрес вашего сервера, пароль root и доступные порты (иногда порты не отображаются, как в моём случае).</b>

<img width="457" height="443" alt="image" src="https://github.com/user-attachments/assets/edcd1a69-fcec-4b49-9c98-fe26155315b3" />

<b>Следующий шаг — открыть консоль сервера. В большинстве случаев консоль уже встроена в панель управления вашего хостинг-провайдера. Ищите раздел «Консоль», «Веб-консоль» или подобный вариант.

Однако в моём случае я буду использовать встроенную командную строку Windows (CMD) и подключаться к VPS-серверу через SSH.
Шаги одинаковы независимо от того, какую консоль вы используете.</b>

# Шаг 2: Настройка консоли и удалённое подключение к VPS-серверу

<b>Откройте консоль, предоставленную вашим хостинг-провайдером, или откройте командную строку Windows (CMD) от имени администратора, как я сделаю в этом руководстве.</b>

## Подключение к VPS-серверу с помощью CMD

Вы увидите окно командной строки, подобное этому.

<img width="959" height="434" alt="image" src="https://github.com/user-attachments/assets/edbf4b3c-7ac9-470e-978d-dae3b42d83bd" />

<b>Для удалённого подключения к серверу мы будем использовать команду `ssh`.

Введите следующую команду в командную строку:</b>

```bash
ssh root@IP_АДРЕС_ВАШЕГО_СЕРВЕРА

```
Команда должна выглядеть примерно так:

```bash
ssh root@185.221.160.4
```

## Использование веб-консоли хостинг-провайдера

<b>В этом случае SSH-подключение будет установлено автоматически. Вам не нужно вводить команду SSH вручную, в отличие от использования CMD.</b>

# Шаг 3: Вход и ввод пароля root

После ввода команды через CMD или открытия веб-консоли вам будет предложено ввести **пароль root**.

В моём случае пароль доступен на странице информации о сервере, поэтому я могу просто **скопировать и вставить** его в консоль.

> **Важно:** Сочетание клавиш для вставки отличается в разных консолях. Во многих консолях на базе Linux **Ctrl + V** не работает. Вместо этого вставьте пароль, **щелкнув правой кнопкой мыши** внутри окна консоли.

> **Примечание:** Если пароль не отображается при вводе, это совершенно нормально. Linux **не** отображает никаких символов (даже точек или звёздочек) при вводе пароля. Просто введите или вставьте пароль и нажмите **Enter**.

Если вход всё ещё не удаётся, попробуйте **ввести пароль вручную** вместо копирования и вставки.

# Шаг 4: Обновление системы и установка Python

<b>Выполните следующие команды по очереди, чтобы обновить систему и установить последние пакеты Python:</b>

```bash
# Обновление списков пакетов
sudo apt update

# Обновление установленных пакетов
sudo apt upgrade -y

# Установка Python и необходимых инструментов
sudo apt install -y python3 python3-pip python3-venv

# Установка Node.js и npm
sudo apt install -y nodejs npm

# Глобальная установка PM2
sudo npm install -g pm2

# Удаление ненужных пакетов
sudo apt autoremove -y

# Проверка установленных версий
python3 --version
pip3 --version
node --version
npm --version
pm2 --version
```

После установки FileZilla запустите приложение. Вы увидите окно, похожее на показанное ниже:

Затем на панели **Локальный сайт** либо введите путь к папке проекта вашего Telegram бота, либо используйте встроенный файловый менеджер, чтобы найти её вручную.

<b>После того как вы нашли папку с ботом, пришло время подключиться к VPS-серверу через FileZilla.

### Перед подключением рекомендуется закрыть сессию консоли, которая использовалась для предыдущего SSH-подключения.</b>

<b>В верхней части окна FileZilla вы увидите следующие поля: **Хост**, **Имя пользователя**, **Пароль** и **Порт**.

Заполните эти поля следующим образом:

- **Хост** — введите IP-адрес вашего VPS-сервера.
- **Имя пользователя** — введите `root` (**важно: используйте строчные буквы**).
- **Пароль** — введите пароль root со страницы информации о VPS-сервере.
- **Порт** — это поле важно. В некоторых случаях порт не предоставляется хостинг-провайдером. Стандартный порт SSH обычно `22` (иногда `21` используется для FTP, но для SSH-подключений это почти всегда `22`).

После заполнения всех полей нажмите **Быстрое подключение** и подождите установки соединения.

Справа вы должны увидеть папку **root**. Это означает, что вы успешно подключились к вашему VPS-серверу.

# Шаг 7: Запуск бота на VPS-сервере

<b>После загрузки файлов на сервер с помощью FileZilla откройте консоль и подключитесь к VPS-серверу так же, как мы делали ранее в **Шаге 2**.</b>

Сначала проверьте, что папка с ботом была загружена правильно, с помощью команды `ls`:

```bash
ls
```

Вывод должен выглядеть примерно так:

```bash
root@ИмяСервера:~# ls
folder_bot
```

Если вы видите имя папки вашего бота, значит, файлы были загружены успешно, и вы можете переходить к следующим шагам.

Если всё правильно, введите следующую команду, чтобы открыть папку с ботом:

```bash
cd ИМЯ_ПАПКИ
```

Замените `ИМЯ_ПАПКИ` на имя папки вашего бота.

Пример:

```bash
cd folder_bot
```

После выполнения этой команды вы перейдёте в каталог проекта вашего бота.

После выполнения команды `cd` путь в терминале должен измениться на каталог вашего бота.

Пример:

```bash
root@ИмяСервера:~/folder_bot#
```

Это означает, что вы находитесь внутри папки проекта бота и можете продолжать следующие шаги настройки.

Затем выполните следующие команды:

```bash

# Проверка файлов в текущем каталоге

ls



# Установка всех необходимых зависимостей Python

pip3 install -r requirements.txt



# Запуск бота с помощью PM2

pm2 start main.py --interpreter python3

```

Замените `main.py` на имя основного стартового файла вашего бота, если оно отличается.

После выполнения этих команд ваш Telegram бот должен запуститься на VPS-сервере.

Чтобы проверить, работает ли процесс бота, используйте следующую команду:

```bash
pm2 list
```
Если бот запущен успешно, вы увидите ваш процесс в списке со статусом **online**.

Пример:

```bash
┌────┬──────────────┬─────────┬────────┐
│ id │ name         │ status  │ mode   │
├────┼──────────────┼─────────┼────────┤
│ 0  │ main         │ online  │ fork   │
└────┴──────────────┴─────────┴────────┘
```
# Заключение

Вот и всё! Ваш Telegram бот успешно развёрнут и работает на вашем VPS-сервере.

Вы выполнили все необходимые шаги:
- Настроили VPS-сервер
- Подключились к серверу по SSH
- Загрузили файлы бота с помощью FileZilla
- Установили необходимые зависимости
- Запустили бота с помощью PM2

Теперь ваш бот должен работать непрерывно в фоновом режиме.
